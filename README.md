# SQL Data Analysis Project

## Project Overview
This project demonstrates SQL data analysis techniques using a retail database. 
The goal is to practice and showcase SQL skills including filtering, aggregations, joins, subqueries, and set operations.

The queries are organized into multiple levels based on complexity.

## Database Tables
The database includes the following tables:

- customers
- products
- orders
- order_items
- payments
- product_reviews

## SQL Concepts Covered

### Level 1: Basics
Basic queries for selecting and filtering data.

Examples:
- Selecting specific columns
- Using DISTINCT
- Filtering using WHERE
- Sorting using ORDER BY
- Using BETWEEN and IN operators

### Level 2: Filtering and Formatting
Queries focused on formatting and data manipulation.

Examples:
- Column aliasing
- CONCAT function
- Date extraction
- Calculated columns

### Level 3: Aggregations
Using aggregate functions for analysis.

Examples:
- COUNT()
- SUM()
- AVG()
- GROUP BY
- ORDER BY

### Level 4: Multi-Table Queries (Joins)
Combining multiple tables to analyze relationships.

Examples:
- INNER JOIN
- LEFT JOIN
- RIGHT JOIN

### Level 5: Subqueries
Using nested queries for advanced filtering.

Examples:
- Comparing values with averages
- Finding maximum values per customer
- Filtering using subqueries

### Level 6: Set Operations
Using multiple conditions to combine results.

Examples:
- Customers who placed orders
- Customers who reviewed products
- Combining results using logical conditions
